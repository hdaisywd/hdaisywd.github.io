# [네트워크] DNS란 무엇인가

📝 **들어가기 전에...**

어느날 구글에 무언가 검색하고 싶어졌다. 그래서 
```
www.google.com
```

을 주소창에 입력한다고 치자. 그러면 우리는 구글 웹사이트에 들어가게 된다. 그러나 사실 인터넷은 이렇게 간단하게 작동하지 않는다. 

![](https://i.imgur.com/s4U8aVc.png)
실제로는 위의 사진과 같이 작동을 하는데, 이 사진은 [TCP School](https://tcpschool.com/webbasic/works)에서 퍼온 것이다. 

>①② 사용자가 웹 브라우저를 통해 찾고 싶은 웹 페이지의 URL 주소를 입력함.
>
③ 사용자가 입력한 URL 주소 중에서 도메인 네임(domain name) 부분을 DNS 서버에서 검색함.
>
④ DNS 서버에서 해당 도메인 네임에 해당하는 IP 주소를 찾아 사용자가 입력한 URL 정보와 함께 전달함.
>
⑤⑥ 웹 페이지 URL 정보와 전달받은 IP 주소는 HTTP 프로토콜을 사용하여 HTTP 요청 메시지를 생성함.
>
이렇게 생성된 HTTP 요청 메시지는 TCP 프로토콜을 사용하여 인터넷을 거쳐 해당 IP 주소의 컴퓨터로 전송됨.
>
⑦ 이렇게 도착한 HTTP 요청 메시지는 HTTP 프로토콜을 사용하여 웹 페이지 URL 정보로 변환됨.
>
⑧ 웹 서버는 도착한 웹 페이지 URL 정보에 해당하는 데이터를 검색함.
>
⑨⑩ 검색된 웹 페이지 데이터는 또 다시 HTTP 프로토콜을 사용하여 HTTP 응답 메시지를 생성함.
>
이렇게 생성된 HTTP 응답 메시지는 TCP 프로토콜을 사용하여 인터넷을 거쳐 원래 컴퓨터로 전송됨.
>
⑪ 도착한 HTTP 응답 메시지는 HTTP 프로토콜을 사용하여 웹 페이지 데이터로 변환됨.
>
⑫ 변환된 웹 페이지 데이터는 웹 브라우저에 의해 출력되어 사용자가 볼 수 있게 됨.

위의 글 또한 퍼온 것인데, 읽고 이해를 해보도록 하자. 이 글은 다양한 계층이 포함된 프로토콜들의 동작 원리가 아닌 DNS의 운용 방식에 대해 설명을 해보려 한다. 

# 1. DNS란?

DNS란 Domain Name System의 약자로 **도메인 이름을 IP로 변환해주는 역할**을 한다.

사용자가 www.google.com 이라고 입력을 하면 DNS는 이를 142.250.199.78로 바꿔준다. 
***(참고: MacOS를 사용하는 경우 터미널에 dig google.com을 쳐보삼. 그럼 ip 주소가 나온다.)***

![](https://i.imgur.com/hlo6y1p.png)

위와 같이 작동하게 되는 것이다. 주소창에 위의 ip 주소를 입력하면 바로 구글이 연결이 된다. (ip가 바뀌면 연결 안 될 수도... ) 그런데 왜 굳이 네임 서버를 따로 쓰냐 하면 우리는 숫자를 기억하지 못하기 때문이다. 
**전화번호부와 똑같다.** 친구들의 전화번호를 언제 다 일일이 기억하고 있겠는가. 그래서 도메인 이름과 ip 주소를 연결해주는 도메인 이름 시스템을 만들어냈고, 그게 바로 DNS다. 

그런데 www.google.com의 ip 주소를 우리의 컴퓨터는 어떻게 알 수 있을까? 도메인 요청이 ip로 바뀌어서 우리에게 돌아오기 까지의 과정을 따라가며 공부해 보도록 하자. 

# 2. DNS의 여정 

📝 **들어가기 전에...**

DNS는 **분산형 데이터 베이스**를 사용한다. 
![](https://i.imgur.com/BZ8UEp4.png)

위의 사진과 같은 구조로 되어있는데 Root DNS 서버, TLD(최상위 도메인) DNS 서버, Authoritative DNS 서버가 있다. 그럼 이제 본격적으로 우리의 요청이 어떻게 처리되는지 알아보자. 

![](https://i.imgur.com/68aSCpQ.png)

## 1) Local DNS 서버

먼저 우리는 로컬 서버에 도메인 이름을 전달한다. 각 통신사별로 DNS 서버가 존재하는데, 캐싱이 되어있는 경우 바로 IP 주소를 반환해준다. 물론 구글 정도면 당연히 캐싱이 되어있을 것 같지만, 만약 없다면?

## 2) Root DNS 서버

우리는 루트 DNS 서버로 가서 요청을 처리하게 된다. 루트 서버는 전 세계에 13군데 있다. 
아무튼 루트 서버에서는 도메인 이름을 보고 .com을 확인 후 .com DNS 서버로 연결을 해준다. 

## 3) .com 서버

이제 .com 까지는 왔다. .com에서는 도메인 이름을 보고 google.com의 서버로 연결을 해준다. 

## 4) google 서버

드디어 google에 도착했다. 여기에는 호스트별 IP 주소가 존재한다. 

![](https://i.imgur.com/0wYrcZH.png)


호스트 이름은 앞 부분! 도메인 이름이 뒷 부분이다. 
아무튼! 구글 서버에 들어가면 각 호스트별 IP 주소가 존재한다. 이 중 우리가 입력한 www.google.com의 ip 주소를 최종 반환해주게 된다.
