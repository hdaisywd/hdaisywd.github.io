# 1. Physical Layer

## 요약

- 0과 1의 나열을 아날로그 신호로 바꾸어 전선으로 흘려 보낸다 (Encoding)
- 아날로그 신호가 들어오면 0과 1의 나열로 해석한다. (Decoding)
- 인코딩과 디코딩을 통하여 물리적으로 연결된 두 대의 컴퓨터가 0과 1의 나열을 주고받을 수 있게 해주는 모듈이다. (Module)

**어디에?** 

- PHY 칩 (1계층 모듈은 하드웨어적으로 구현되어 있다. 즉, 회로!)

# 2. Data Link

**들어가기 전에…**

컴퓨터를 서로서로 전부 연결한다면? → 비용이 너무 많이 든다! 

![image](https://github.com/hdaisywd/bean-noodle/assets/102342953/c4f61b00-3500-464d-bf25-89071e4c92f2)

그렇다면 이런씩으로 한번에 연결을 해 줄 수 있다. 그런데, 목적지를 알 수 없다면 한 PC에서 보낸 신호를 모든 PC가 수신하게 될 것이다. 

따라서 **목적지를 알 수 있게 해주는 장치인 스위치(Switch)**가 등장하게 된다. 

그렇다면 우리는 이제 한 스위치에 연결된 PC가 아니어도 서로 통신할 수 있으려면 어떻게 해야할까? 스위치 끼리 서로 연결을 해주면 된다. 

![image](https://github.com/hdaisywd/bean-noodle/assets/102342953/af3299e9-22c0-4434-8e5b-5783bb4b8d6b)

그리고 이 **스위치들이 서로 연결되어 서로 다른 네트워크(인트라넷)에 속한 PC 끼리 통신이 가능**하게 해주는 장비를 **라우터(Router)**라고 한다. 

요런씩으로 위로 위로 올라가다보면 인터넷이 나온다. 전세계의 PC는 서로 이런씩으로 데이터를 주고 받게 된다. 즉, **전 세계의 컴퓨터들이 계층구조로 연결**되어 있는 것이 **인터넷**인 것이다. 

**그렇다면 여러대의 컴퓨터가 통신하려면?**

여러대의 컴퓨터가 동시에 한 컴퓨터에게 데이터를 보낸다고 생각해보자. 

PC1 / 0101

PC2 / 1110

만약 특별한 조치가 없다면 PC3가 받는 데이터는 01011110이 될 것이다. 어디서부터 어디까지 끊어 읽어야 하는지 알 수 없게 되는 것이다. 

이를 위해 **송신자는 데이터의 앞 뒤에 특정한 비트열을 붙이게 된다.** 

이를 프레이밍(Framing)이라고 부른다. 

## 요약

- 같은 네트워크에 있는 여러 대의 컴퓨터들이 데이터를 주고 받기 위해서 필요한 모듈 (Module)

**어디에?**

- 랜(LAN) 카드 (2계층 또한 하드웨어로 구성된 모듈이다.)

# 3. Network Layer

**들어가기 전에…**

![image](https://github.com/hdaisywd/bean-noodle/assets/102342953/af3299e9-22c0-4434-8e5b-5783bb4b8d6b)
맨 왼쪽의 PC를 PC1, 맨 오른쪽의 PC를 PC6이라고 해보자. 1번에서 6번으로 데이터를 보내기 위해서는 6번의 IP 주소를 알아야 한다. **IP 주소 + data** 를 담아서 신호를 보내게 된다. 

즉, 데이터를 보내기 위해서는 **IP 주소를 필수적으로 알아야**한다는 것이다.

IP 주소 + data는 **패킷(Packet)**이라고 부른다. 

**패킷의 여정**

PC1이 패킷을 스위치에 보내게 되면 data와 ip주소를 분리해서 해석하게 된다. 그런데 본인 휘하의 컴퓨터 중에는 해당 주소를 가진 컴퓨터가 없을 것이다. 따라서 스위치는 다시 포장을 한 다음 그 위의 라우터에 다시 패킷을 보내게 된다. 그리고 라우터에 다시 한 번 데이터와 ip주소를 해석해서 보내야 할 곳을 알게 된다. (이는 **라우팅**이라고 한다. 추후 공부해보면 좋을듯!) 이렇게 쭉쭉 가다가 주소를 발견하면 PC6에 패킷을 전달하게 되는 것이다.

```
여기서 잠깐! IP? MAC?
    
MAC 주소란?
    
- 네트워크 통신에서 통신 기기의 식별번호이다.
- 세상에 단 하나밖에 없는 유니크한 값을 가지며 변경될 수 없다.
- ARP(Address Resolution Protocol)는 장치의 MAC 주소를 수신하는데 사용되는 프로토콜이다.
    
라우팅 기법은 패킷에 포함된 ip 주소를 추적해서 최단 경로를 선택해 가게 되어 있다. ip 주소는 논리적인 주소이기 떄문에 ip에서 ip 주소로 목표된 네트워크로 추적해서 패킷을 보낸 뒤 그곳에서 ip에 등록된 mac 주소로 주소가 변환되어 그 컴퓨터로 패킷을 전송 하게 된다. 
    
즉, ip는 라우팅하는 주소로 실질적인 통신X
```

 

## 요약

- Inter-network 속에서 목적지 컴퓨터로 데이터를 전송하기 위해 IP 주소를 이용해서 길을 찾고 (라우팅, routing) 자신 다음의 라우터에게 데이터를 넘겨주는 (포워딩, forwarding) 모듈이다.

**어디에?**

- 운영체제의 커널에 소프트웨어 적으로 구현되어 있다.

```
커널이란?
    
컴퓨터 과학에서 커널(kernel)은 **컴퓨터 운영 체제의 핵심이 되는 컴퓨터 프로그램**으로, 시스템의 모든 것을 완전히 제어(control)한다. 운영 체제의 다른 부분 및 응용 프로그램 수행에 필요한 여러 가지 서비스를 제공한다. 핵심(核心)이라고도 한다.
```

    

# 4. Transport Layer

**들어가기 전에…**

컴퓨터에서는 이제 데이터를 받게 되었다. 그런데 컴퓨터에서는 다양한 프로그램들이 실행되고 있을 것이다. 그렇다면 어떤 프로세스에게 데이터를 넘겨줘야 하는지 어떻게 알 수 있을까?

프로세스들은 **포트 번호**를 가져야 한다. 포트 번호란 **하나의 컴퓨터에서 동시에 실행되고 있는 프로세스들이 서로 겹치지 않게 가져야하는 정수값**이다.

송신자는 데이터를 보낼 때 데이터를 받을 수신자 컴퓨터에 있는 프로세스의 포트 번호를 붙여서 보낸다. 

## 요약

- 포트 번호를 사용하여 도착지 컴퓨터의 최종 도착지인 프로세스 까지 데이터가 도달하게 하는 모듈 (Module)
- Encoding 과정에서는 포트 번호를 붙여주게 되고 Decoding 과정에서는 포트 번호를 보고 알맞은 프로세스에 데이터를 전달하게 된다.

**어디에?**

- 운영체제의 커널에 소프트웨어적으로 구현되어 있다.

# 5. Application Layer

**들어가기 전에…**

Presentation, Session 레이어는 어디감..?

→ 현대의 컴퓨터는 OSI 모델이 아닌 TCP/IP 모델을 따르고 있다. 

![image](https://github.com/hdaisywd/bean-noodle/assets/102342953/5539fbb5-7add-4cfd-94e5-c6ae535cbd02)

**TCP/IP 소켓 프로그래밍(네트워크 프로그래밍)**

운영체제의 Transport Layer에서 제공하는 API를 활용해서 통신 가능한 프로그램을 만드는 것

- 소켓 프로그래밍 만으로 클라이언트, 서버 프로그램을 따로따로 만들어서 동작시킬 수 있다.
- 또한 누구나 자신만의 Application Layer 인코더와 디코더를 만들 수 있다.
- 즉, 누구나 자신만의 Application Layer 프로토콜을 만들 수 있는 것이다.

**대표 프로토콜 HTTP**

먼저 **클라이언트 & 서버 패러다임**을 알아야합니다! 

- 클라이언트 & 서버 패러다임
    1. **서버** 
    - 항상 켜져 있다
    - IP 주소가 영구적이다
    2. **클라이언트** 
    - 서버와 연결하고 소통한다
    - 간헐적으로 연결된다
    - 유동적인 ip 주소를 가지고 있다.
    - 클라이언트 끼리 바로 communicate를 하지 않는다.

![image](https://github.com/hdaisywd/bean-noodle/assets/102342953/404aea12-772e-4feb-a2da-794b3553885f)

→ MVC 패턴과 마찬가지로 소프트웨어 아키텍처 중에 **Layered Architecture**라는 것이 있고, 이를 따르는 대표적인 예가 **네트워크 시스템**이다. 

즉, 네트워크 시스템 == 하나의 커다란 소프트웨어. 

OSI 7 Layer 모델은 거대한 소프트웨어의 구조를 설명하는 것!
