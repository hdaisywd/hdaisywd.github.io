# 2. OS의 역사 

## 1세대 운영체제 (1940 ~ 1950년대)

### 일괄처리 시스템(Single-Stream Batch Processing Systems)의 등장 

**Single-Stream이란?**
- 작업은 차례대로 한 개씩 처리가 된다. 즉, **한 개의 작업이 메모리에 올라가 시작되면 그 일이 완전히 끝날 때까지 다른 작업은 기다려야 한다.**
- 작업이 끝날 때까지 **사용자의 중간 개입이 허용되지 않는다.**

**일괄처리란?**
- 다수 개의 프로그램을 읽어 저장해 놓고, 한 번에 한 개씩 프로그램을 실행시켜 주는 방식
- 프로그램들은 천공카드로 만들어지며, 만들어진 프로그램들을 카드 판독기가 읽어 저장 
- 카드 판독기는 천공카드들을 수직으로 쌓아 놓고 빠르게 읽음 


## 2세대 운영체제 (1960년대 전반기)

### 트랜지스터 컴퓨터 등장  
### 다중 프로그래밍 시스템(Multiprogramming System)

- 프로그램이 실행되어지기 위해서는 반드시 Main memory에 있어야 한다. (폰 노이만의 stored program concept)
- 다중 프로그래밍 시스템이란 CPU를 여러 프로그램이 번갈아서 사용할 수 있도록 한다. 
- 이때 하드 디스크에 프로그램이 올라가 있다면 시간이 낭비되게 된다. 따라서 메인 메모리에 여러개의 프로그램을 올려놓도록 한다. 

### 다중 처리 시스템(Multiprocessing System)

- 다중처리 시스템이란 CPU를 여러개를 두는 것이다. 
- 다중 프로그래밍 시스템이 구현되어 있어야 다중처리 시스템이 구현될 수 있다. 

### 시분할 시스템(Timesharing System)

- 만약 메모리에 여러개의 프로그램을 올려놓고 하나하나 처리하면 Batch System과 다를게 없다. 
- 따라서 CPU가 처리해줄 수 있는 시간을 작업 수에 맞춰 분할하여 각자에게 일정량만큼씩 분배하여 번갈아 처리하도록 하는 것이 시분할 시스템이다. 
- 시분할 시스템 또한 다중 프로그램 시스템이 먼저 선행되어야 한다. 

### 대화식 시스템

- 시스템과 사용자가 모니터나 입력 장치를 통해 대화하듯이 일을 처리해 나가는 방식을 대화식 시스템이라고 한다. 
- 즉, 중간개입이 가능하다.

# 3. OS 안에는 어떤것들이 있을까?

![](https://i.imgur.com/Ecn5sPH.png)

**쉘(Shell)**  
- UNIX전에는 Command Interpreter라고 불렸다. UNIX에서의 Command Interpreter의 이름이다. 
- 사용자 명령을 받아 실행을 한다.  
**장치관리**  
- 시스템에 있는 주변 장치(키보드, 모니터, 프린터, 하드디스크) 제어한다.  
**파일 관리**  
- 사용자 또는 프로그래머로 하여금 파일을 만들고, 지우고, 수정하고 이름을 부여하며 디렉토리 등을 관리한다.  
**메모리 관리**  
- 주기억 장치를 관리한다. 
- 실행될 프로그램을 위해 주기억 장치의 일정 부분을 할당하거나 주기억 장치 안에 있는 프로그램들의 경계를 설정하여 서로 침범하지 못하게 하는 등의 일을 처리한다.  
**처리기 관리**  
- 컴퓨터의 처리기 즉, CPU의 처리 능력을 각 작업에 효과적으로 배분되도록 관리한다.  

*이 외에도 구성요소들이 더 있으며, OS에 따라 구분법도 다를 수 있다.*

# 4. OS는 컴퓨터의 어디에 있을까?

운영체제는 **커널(Kernel), 쉘(Shell), 유틸리티 프로그램(Utility Program)**으로 나뉜다. 

**커널(Kernel)**
- 운영체제의 각종 기능들 중 사용자와 실행 프로그램을 위해 매우 빈번하게 사용되는 부분이다. 
- 컴퓨터가 처음 부팅 될 때에 주기억 장치에 적재되어 시스템의 운영이 종료 될 때까지 계속해서 주기억 장치에 남아 있게 되는 부분이다. 
- 운영체제의 핵심이며 핵, 관리자, 메모리 상주 프로그램(Memory Resident Program)이라고도 불린다. 

### 듀얼 모드(Dual Mode)란?

- 대부분의 컴퓨터 시스템에는 두 개의 모드 User Mode와 Kernal Mode를 정해놓고 각각의 프로그램은 이중 하나의 모드에서 실행되도록 한다. 
- **커널 모드: 운영체제가 실행되는 모드, 모든 명령어 수행 가능**
- **유저 모드: 사용자 프로그램, 응용 프로그램 등이 실행되는 모드**
- 목적은 시스템의 보호와 장치들의 효율적인 관리이다. 

**시스템 호출(System Call)이란?**
- 사용자 프로그램이 운영체제의 어떤 서비스를 요구할 떄 커널을 호출하는 것
- 운영체제는 사용자가 요구한 서비스를 커널 모드에서 실행한 후, 사용자 모드에서 사용자 프로그램이 실행되도록 리턴한다.
