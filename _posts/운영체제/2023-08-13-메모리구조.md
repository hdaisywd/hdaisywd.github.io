프로그램을 실행할 때 어떤씩으로 요청이 들어가는지 확인해볼까요?

![image](https://github.com/hdaisywd/hdaisywd/assets/102342953/90efc89b-5a46-449a-967b-3db3fae4f3d2)

위의 그림과 같이 진행된답니다. 여기서 우리가 오늘 공부할 부분은 RAM입니다. 

> RAM이란? <br>
> Random Access Memory는 어플리케이션에서 단지거으로 데이터 저징 및 엑세스 공간을 제공하는 컴퓨터의 임시 저장소입니다. 

그럼 이제 본격적으로 램의 메모리 구조에 대해 배워봅시다. 

![image](https://github.com/hdaisywd/hdaisywd/assets/102342953/3adf664b-f3a5-495e-b1eb-4b00da111da7)

RAM의 메모리 구조는 크게 코드 영역, 데이터 영역, 힙 영역, 그리고 스택 영역으로 나뉩니다. 

## 1. 코드 영역

코드 영역은 쉽게 말해 TEXT 영역이라고 이해하시면 됩니다. 우리가 작성한 코드가 컴파일러를 통해 기계어로 번역이 되어 기게어 형태로 코드 영역에 저장되게 됩니다. 실행 파일을 구성하는 명령어들이 올라가게 됩니다. 

## 2. 데이터 영역

두 번째로 데이터 영역입니다. 이 곳에는 전역변수, static 변수, 문자열 등이 저장됩니다. 프로그램의 시작과 동시에 할당되고 프로그램이 종료되면 그제서야 메모리에서 소멸됩니다. 

## 3. 힙 영역

힙 영역입니다. 여기서부터 어디서 많이 들어봄.. 힙 영역은 동적으로 할당된 변수가 할당되는 곳입니다. 메모리가 위쪽 주소부터 할당됩니다. FIFO 구조란뜻. 크기가 런타임에 결정이 됩니다. 따라서 메모리 해제를 해주지 않으면 메모리 누수의 위험이 있는 영역입니다. c에서는 free, c++에서는 delete, 자바는 가비지 컬렉터등이 이 문젤르 해결해줍니다. swift에서는 클래스 인스턴스, 클로저 같은 참조 타입이 이 부분에 해당됩니다. ARC를 통해 자동으로 해제도 시켜줍니다. [ARC란?](https://hdaisywd.github.io/appdev/TIL0720/)

```swift
class Person {
    var name: String?
    var age: Int?
}

var person: Person = Person()
```

클래스 인스턴스는 힙에, person 변수는 스택 영역에 할당됩니다. 

## 4. 스택 영역 

컴파일 타임에 크기가 결정됩니다. 지역 변수, 매개 변수 등이 저장됩니다. 스택은 LIFO 형식입니다. 함수가 호출되면 매개변수, 지역변수가 저장되게 되면 함수 호출이 끝날 시 변수들이 pop 되어 사용중인 메모리 공간이 비워지게 됩니다. 스택 오버플로우가 이곳에서 나온 얘긴데, 예를 들면 이렇습니다. 재귀 함수가 끝나지 않아 계속해서 부르다 보면 지정된 스택의 크기를 넘어서서 넘쳐나게 되겠죠?! 그래서 오버플로우가 납니다. 스택 영역은 이처럼 조심해서 다뤄주지 않으면 보안 문제와 같은 문제가 생기기 쉬운 곳입니다.
