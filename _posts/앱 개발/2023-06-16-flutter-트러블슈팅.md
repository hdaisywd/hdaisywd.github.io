# flutter binary not on your path 해결 ⚙️

<img width="950" alt="Screenshot 2023-06-18 at 2 20 36 PM" src="https://github.com/hdaisywd/todo_diary/assets/102342953/24d8eaa3-0c4a-429a-bcb6-1d2e84a45bc3">

또.. 또 에러가 뜬다. 일단 안내 사항에 따라보자. 

```
sudo chown -R $ (whoami) /Users/daisy/.config/flutter/tool_state
```

입력해주고 flutter doctor를 vscode > view > flutter doctor로 다시 돌려보자! 

<img width="936" alt="Screenshot 2023-06-18 at 2 48 26 PM" src="https://github.com/hdaisywd/todo_diary/assets/102342953/7c739913-e0a4-43d5-8721-fa850fb75738">

오케이? 저게 문젠가보다. 또 안내사항을 따라가 보자. 

<img width="641" alt="Screenshot 2023-06-18 at 3 03 20 PM" src="https://github.com/hdaisywd/todo_diary/assets/102342953/d636c221-f3b5-4604-8f3b-972689d1154d">

난 제대로 다 path 설정을 해줬다! 

<img width="661" alt="Screenshot 2023-06-18 at 3 25 50 PM" src="https://github.com/hdaisywd/todo_diary/assets/102342953/012be8e1-b482-48e2-b39a-c80f634699ec">

이것도 해줬는데.. source ~/.zshrc를 해주려니 에러가 떠서 xcode 커멘드 라인을 지웠다가 다시 깔아줘봤다. 그래도 안된다. 뭐가 문젤까... 

[해결을 위해 참고한 스택오버플로우 링크](https://stackoverflow.com/questions/33486633/upgrading-oh-my-zsh-gives-me-not-a-git-repository-error) 에 나온 그대로 따라가줬다. 

```
cd ~/.oh-my-zsh
git init # you've already done this step, so you can skip
git remote add origin https://github.com/ohmyzsh/ohmyzsh.git
git fetch
git reset --hard origin/master
```

git 레포가 remove 되는 바람에.. 생긴 에러! 이제 omz 업데이트는 정상적으로 되고, source ~/.zshrc도 정상적으로 작동한다. 

드디어 path는 인식을 하기 시작했다. 그런데, 또 문제가 생겼다.
