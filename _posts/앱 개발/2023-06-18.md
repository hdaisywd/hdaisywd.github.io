Flutter는 Dart 언어로 쓰여진 프레임워크(Framework)로, 어떠한 규칙과 메뉴얼대로 코드를 작성하면 앱/웹 서비스를 만들어주는 틀이다. 나는 VSCode를 이용해서 flutter 개발을 해볼 것이다. 

# 1. Flutter 장점 👩‍💻

## 1) 앱 개발 방법 

1. 네이티브 앱 
    - Android Studio, iOS SDK를 이용해서 개발. 다른 언어로 두 개의 앱을 개발해야 한다. (다만, 네이티브 앱으로 개발을 하면 성능이 가장 빠르다는 장점이 있다. 또한, 최신 기술을 가장 빠르게 적용시킬 수 있다.)
2. 크로스 플랫폼 앱
    - React Native
        - 자바스크립트 기반. 페이스북에서 출시한 오픈 소스 모바일 앱 프레임 워크
    - Flutter
        - 구글에서 출시한 오픈 소스 모바일 앱 프레임 워크
    - 네이티브 앱 보다는 성능이 뒤쳐지지만 하나의 언어로 안드로이드, iOS에서 모두 실행되는 앱을 빠르게 만들 수 있다. 

## 2) 그렇다면 왜 Flutter 인가

1. 크로스 플랫폼이어서 생산성이 월등하다. 
2. React Native보다 훨씬 핫하다. 
3. 공식 문서가 굉장히 잘 돼 있다. 
4. 성능이 뛰어나다. (연산 시간이 월등하다.)

# 2. Flutter와 위젯 👩‍💻

Flutter는 모든 것이 위젯(widget)으로 이루어져 있다. 위젯은 레고 블럭과 같이 앱을 만드는데 사용되는 작은 모듈이다. 

![Untitled](https://github.com/hdaisywd/todo_diary/assets/102342953/6457d513-7212-4a9e-9b2a-044cbba873e9)

이런 위젯들은 자녀 위젯들을 가지는 식으로 구조가 그려지기 때문에 위젯 트리라고 부른다. 즉, 상단의 위젯으로 부터 자녀 위잿들이 계속해서 가지치기를 하게 된다. 

[위잿 카탈로그(Widget catalog)](https://docs.flutter.dev/ui/widgets) 페이지를 눌러보자. Flutter에서 사용하는 위젯을 모아둔 페이지다.

# 3. 프로젝트 준비 👩‍💻

## 1) Flutter 폴더 만들기

어디에도 만들어줘도 되지만 중요한 것은 **플러터 프로젝트 경로에 한글이 오면 절대 안된다!!** 특히 user이름 한국어로 해둔 분들.. 주의하십셔. 

나는 바탕화면에 만들었다. 이제 VSCode로 진행해보자. 우선 Extension에서 flutter를 install 해주자. 나는 확장팩을 다운 받아 준 이후에도 sdk를 찾을 수 없다길래 수동으로 sdk 파일의 위치를 설정해주었다. 

이제 view > command palette > flutter:new project > application을 클릭해주고, 바탕화면의 flutter 폴더에 프로젝트를 생성해주자. 

<img width="925" alt="Screenshot 2023-06-18 at 1 04 20 PM" src="https://github.com/hdaisywd/todo_diary/assets/102342953/1d827000-d895-4bec-8835-792e6d527aad">

이름에는 언더바(_)와 영어 소문자만 들어갈 수 있으니, 유의하자. 

## 2) 디렉토리 이해 및 설정 

> lib: 주로 코딩하는 폴더
> pubspec.yaml: 라이브러리 및 설정을 하는 폴더
> android, ios: Android, iOS 프로젝트 폴더
> web: Web 프로젝트 폴더

이중 main.dart의 내용을 수정해보자. 

우선 기존의 내용을 모두 지우고 아래의 코드를 복사 붙여넣기 하자. 

```dart
import 'package:flutter/material.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  const MyApp({Key? key}) : super(key: key);

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      debugShowCheckedModeBanner: false,
      home: Scaffold(
        appBar: AppBar(),
      ),
    );
  }
}
```

다음으로 analysis_options.yaml의 rules에 밑의 코드를 추가해보자. 

```yaml
    prefer_const_constructors: false
    prefer_const_literals_to_create_immutables: false
```

지금 main.dart를 열면 파란색 실선이 뜨는데, 이를 없애기 위한 코드이다. 아직 초보자인 입장에서 가독성을 높여주기 위한 것으로, 코드 상의 큰 의미는 없다. 
파란 실선은 개선할 여지가 있는 코드 밑에 뜨는데, const라는 키워드를 붙여 상수로 선언하라는 제안이고, 우린 이 옵션을 잠시 꺼둘 것이다. 

마지막으로 view > command palette > dart use recommended settings를 눌러 저장 시 자동 줄 정렬 기능과 같은 편의 기능 설정을 적용해보자.
