# Flutter란? 

> Flutter transforms the app development process. Build, test, and deploy beautiful mobile, web, desktop, and embedded apps from a single database.

Flutter 공식 홈페이지에 나와있는 소개 문구다. 쉽게 말해, Flutter란 앱 개발을 도와주는 프레임워크란 뜻이다. 구글에서 개발한 프레임워크이며 프론트엔드와 백엔드를 모두 개발할 수 있는 풀스택 프레임워크다. 처음 출시됐을 때에는 앱 개발만 지원이 됐지만, 이젠 iOS, Android, 웹, Windows, MacOS, Linux 플랫폼에 대한 앱 개발도 지원한다. 이 외에도 Flutter는 구글에서 개발한 언어인 Dart를 사용한다. Dart는 UI 구축에 최적화되어있다. 

[Flutter에 대해 더 알아보기!](https://aws.amazon.com/ko/what-is/flutter/)

Flutter를 활용해주기 위해 우리가 설치해야 하는 건 크게 flutter, visual studio, android용 프로그램들, iOS용 프로그램들이 있다. 나는 컴퓨터 용량도 부족하고 iOS 개발 전 flutter를 써보는 것이니 iOS 관련된 프로그램만 깔도록 하겠다.

# Flutter 설치하기 

> Flutter는 안드로이드 앱과 iOS 앱을 동시에 개발할 수 있도록 도와주는 프레임워크다! 따라서 Xcode 뿐만 아니라, Android Studio도 깔아주어야 한다. 

## 1. Framework 설치하기 

1. Flutter 설치 

- Flutter 설치 할 폴더 만들기 

맥북 왼쪽 위의 Go > Home 을 클릭해준다. 그럼 사용자 명과 같은 이름의 폴더로 이동되게 된다. 이 곳에 development라는 이름의 폴더를 생성한다. 이 폴더에 flutter 압축 파일을 저장할 것이다. 

[다운 받을 링크](https://docs.flutter.dev/get-started/install/macos)로 들어가 Intel 용 번들을 다운받는다. 그 후, development 폴더로 옮겨준다. 

다운 받은 파일을 실행시켜준다. 그럼 압축 파일이 해제되고 같은 폴더에 flutter라는 폴더가 생성되게 된다. 

이제, flutter 파일로 들어가 bin 폴더를 들어가자. 해당 폴더에는 dart, flutter, dart.bat, flutter.bat이 들어가 있는 것을 확인할 수 있다. 애네는 flutter 실행 파일이다. 이 실행 파일은 현재 해당 폴더 경로에서만 사용할 수 있다. 그러나 컴퓨터 어디에서든 사용할 수 있게 만들어주어야 한다. 이때 필요한게 **환경 변수** 설정이다. 환경 변수에 flutter의 폴더 경료를 추가해주면 컴퓨터 시스템의 어디에서든지 flutter 파일에 접근하고, 실행할 수 있다. 

## 2. 환경 변수 설정 및 설치하기

우선 본인의 macOS 버전을 확인해준다. 사과 모양 > about this mac에 들어가면 확인할 수 있는데, 내 맥은 Ventura 13.1을 사용한다. 가장 최신 것이다. 내가 입력해야 하는 명령어는 다음과 같다. 

```
echo 'export PATH="$PATH:$HOME/development/flutter/bin"' >> ~/.zshrc && source ~/.zshrc
```

본인의 맥 버전을 검색해보고 Catalina 이상이면 위의 명령어를, Mojabe이하면 

```
echo 'export PATH="$PATH:$HOME/development/flutter/bin"' >> ~/.bash_profile && source ~/.bash_profile
```

이 명령어를 이용해주면 된다. 

해당 명령어를 냅다! 입력하기 싫으면 (내가 사용한 방법) 터미널 창에 vi $HOME/.zshrc를 치고, export PATH="$PATH:$HOME/development/flutter/bin 요걸 그대로 입력해주면 된다. 

<img width="512" alt="Screenshot 2023-06-14 at 2 40 27 PM" src="https://github.com/hdaisywd/hdaisywd.github.io/assets/102342953/73cc6f33-b7d9-47f6-9fea-539f4c11cf3c">

이렇게! 그리고 esc -> wq를 입력해주면 저장 완료. 설정을 반영하기 위해 source ~/.zshrc 까지 해주면 진짜 끝끝끝이다.

> 💡 내가 맞닥뜨린 에러 <br>
> exception: flutter failed to create a directory at "/users/daisy/.config/flutter".
> 이때는 sudo를 붙여서 flutter를 실행시켜줘 보면 잘 된다! 관리자 허락이 필요한 작업이라 그런 것 같다. 

```
sudo flutter --version
```

으로 flutter version을 체크해주는 명령어를 처음에 한 번 sudo를 붙여 실행해주고 나면 에러가 뜨지 않는다.

# iOS 프로그램 설치

이제 IDE(프로그래머가 소프트웨어 코드를 효율적으로 개발하도록 돕는 애플리케이션)을 설치할 차례다. 

## 1. Xcode 설치

첫번째로 Xcode! 난 이미 설치해놨으니.. pass.. 한가지 팁은 절대 절대로 App Store에서 깔지 말라는 것이다. 진짜 오래 걸림. 인터넷 홈페이지로 들어가서 수동으로 다운 받고 설치하세요!! 

## 2. Homebrew 설치

난 Homebrew도 이미 설치해놨다.... 따라서 pass... 인터넷을 참고하여 Homebrew를 설치해보도록 하자. Homebrew가 무엇인지 간략하게 소개하자면, 맥에서 소프트웨어를 설치, 삭제하는 것을 도와주는 패키지 관리자이다. 

## 3. CocoaPods 설치

드디어 내 컴퓨터에도 없는 프로그램 등장. CocoaPods는 다른 사람이 만든 코드를 가져올 때 필요한 프로그램이며 Xcode와 함께 iOS 개발에 필요한 프로그램이다. 

```
brew install cocoapods
sudo gem install cocoapods
```

두 개를 터미널에 입력해보자. 참고로 꽤 걸리는게 정상이니 괜찮다. sudo 명령어를 이용하면 비밀번호 입력창이 뜬다. (위에서 나와 같은 에러를 만났으면 이미 해봤을 거다.) 그럼 컴퓨터 비밀번호를 입력하면 된다. 비밀 번호 틀렸으면 cmd + c 누르고 다시 명령어 입력해주면 된다.

Android는 내가 설치를 안 했으니 pass. 근데 Xcode에 문제가 있다고 뜬다. 나 Xcode 설치되어 있는데요... 일단 하라는 대로 해준다. 

```
sudo xcode-select --switch /Applications/Xcode/Contents/Developer
sudo xcodebuild -runFirstLaunch
```

그랬더니 이제 CocoaPods installed but not working 문제만 뜬다. 해결해보자... 

```
sudo gem uninstall cocoapods
sudo gem install cocoapods
```

안된다. 똑같은 에러 또 뜸 ㅎ 다시 구글링을 해보자. 

스택오버플로우를 보니 루비를 이용해서 문제를 해결하는게 보였다. 이걸 이용해 보자.
